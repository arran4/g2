# NAME

g2 - Gentoo overlay and repository maintenance utilities

# SYNOPSIS

**g2** [*COMMAND*] [*SUBCOMMAND*] [*FLAGS*] [*ARGS...*]

# DESCRIPTION

**g2** is a collection of command-line utilities for creating, managing, and working with Gentoo overlay repositories. It provides tools for manipulating `Manifest` files, updating `metadata.xml`, interacting with ebuilds, linting repositories, and generating static HTML sites representing overlays.

The command set supports local overlay maintenance as well as comprehensive tools to index and serve search data.

# COMMANDS

## `manifest`
Commands relating to **Manifest** files.

- **upsert-from-url** *<url>* *<filename>* *<manifestFileOrDir>*
  Downloads a file from *<url>*, calculates required checksums, and updates or inserts the entry into the `Manifest` file.
  Flags: `-blake2b`, `-blake2s`, `-md5`, `-rmd160`, `-sha1`, `-sha256`, `-sha3_256`, `-sha3_512`, `-sha512` (default: `blake2b` and `sha512`).

- **verify** [*--fix*] [*--clean*] *<manifestFileOrDir>*
  Verifies the `Manifest` entries against the ebuild URIs. If `--fix` is passed, missing entries will be downloaded and upserted. If `--clean` is passed, unused entries will be removed.

- **clean** *<manifestFileOrDir>*
  Removes unused entries from the `Manifest` file.

## `metadata`
Commands relating to modifying **metadata.xml** files.

Usage: `g2 metadata [flags] [location]`

- **-l, -longdescription** *<string>*
  Set the long description.
- **-m, -maintainer-add** *<email[:name[:type]]>*
  Add or update a maintainer.
- **-maintainer-remove** *<email>*
  Remove a maintainer.
- **-u, -upstream-add** *<type:id>*
  Add an upstream remote ID (e.g., `github:gentoo/gentoo`).
- **-upstream-remove** *<type:id>*
  Remove an upstream remote ID.
- **-use, -use-add** *<name:description>*
  Add or update a USE flag description.
- **-use-remove** *<name>*
  Remove a USE flag description.
- **-force**
  Force overwrite if type mismatches or other errors occur.

### `metadata discover`
Discover flags and metadata.

## `ebuild`
Commands for manipulating and inspecting **.ebuild** files.

- **init** *<template_name>* *<ebuild_file>*
  Initializes a new ebuild from the specified template.
- **templates**
  Manages ebuild templates.
- **sh-parse-to-json** *<ebuild_file>*
  Parses an ebuild using the shell parser and outputs JSON.
- **as-json** *<ebuild_file>*
  Parses an ebuild natively into JSON format.
- **explain** *<ebuild_file>*
  Output a human-readable summary of an ebuild.
- **query** *<ebuild_file>* `--key` *<key>* [`--format` *lines*]
  Queries specific fields from a parsed ebuild output, rather than dumping the whole JSON.

## `overlay`
Commands for managing a single overlay.

- **ebuild move** *<from>* *<to>*
  Record a package move in `profiles/updates`.
- **ebuild slotmove** *<package>* *<from>* *<to>*
  Record a slot move in `profiles/updates`.
- **ebuild install** [*--category <string>*] *<ebuild.ebuild>* [*overlay_path*] [*-- files...*]
  Installs an ebuild into the overlay, optionally providing files for the `files/` directory. Automatically triggers manifest, cache, use desc, and pkg_desc_index generation.
- **site generate** [*--out <dir>*] [*--clear*] [*location*]
  Generates a static HTML site for the overlay.
  Flags: `-out` (default: `site_out`), `-clear`, `-fast-git-modtime`, `-recent-duration`.
- **license list**
  Lists all configured licenses.
- **license show** *<name>*
  Shows the contents of a license.
- **license add** *<name>* *<file>*
  Adds a new license from a file.
- **license remove** *<name>*
  Removes a license.
- **license alias** [*list|add|remove*]
  Manages license aliases (groups).
- **info-vars** [*list|add|remove*]
  Manages info variables.
- **info-pkgs** [*list|add|remove*]
  Manages info packages.

## `overlays`
Commands for working across multiple overlays.

- **site generate** [*--out <dir>*] [*--clear*] *<repositoriesFile>*
  Generates an aggregated static HTML site for multiple remote repositories described in a `repositories.xml` file.

## `site`
Commands relating to generated static sites.

- **serve** [*--port <int>*] [*path_to_overlay*]
  Serves the generated static site locally for previewing. Defaults to port 8080.

## `lint`
Lints the repository for QA issues and consistency errors.

Usage: `g2 lint [location]`

Checks for missing `md5-cache` files, `IUSE` variables missing from `metadata.xml`, unsupported `layout.conf` properties, and more.

## `use`
Commands for working with **USE flags**, **use.desc**, and **use.local.desc**.

- **discover** [*location*]
  Discovers USE flags from ebuilds to regenerate `use.desc`, `use.local.desc`, and `metadata.xml`.
- **desc-add** *<name>* *<description>*
  Adds a global USE flag description.
- **desc-remove** *<name>*
  Removes a global USE flag description.
- **desc-edit** *<name>* *<description>*
  Edits a global USE flag description.
- **desc-list**
  Lists global USE flag descriptions.
- **local-desc-add** *<pkg>* *<name>* *<description>*
  Adds a local USE flag description for a specific package.
- **local-desc-remove** *<pkg>* *<name>*
  Removes a local USE flag description.
- **local-desc-edit** *<pkg>* *<name>* *<description>*
  Edits a local USE flag description.
- **local-desc-list**
  Lists all local USE flag descriptions.

## `cache`
Commands for managing **md5-dict** cache files.

- **generate** [*location*]
  Generates `md5-dict` cache for ebuilds.
- **verify** [*location*]
  Verifies the cache matches the ebuilds.
- **clean** [*location*]
  Cleans unused cache entries.
- **list-methods**
  Lists available caching methods.
- **set-method** *<method>*
  Sets the active cache method in `layout.conf`.

## `pkg-desc-index`
Commands for maintaining the package description index.

- **generate** [*location*]
  Generates a `pkg_desc_index` file from the repository.
- **verify** [*location*]
  Verifies the existing `pkg_desc_index` matches the repository.

## `package`
Commands relating to package search and indexing.

- **search** [*--path <string>*] *<query>*
  Searches the index for packages matching the query. Advanced queries support boolean logic (AND, OR, NOT), grouping, and Gentoo version ordering (e.g., `version:>1.2.3`).
- **index-overlay**
  Indexes a single overlay for search.
- **index-repositories**
  Indexes multiple repositories from an XML file.
- **index**
  Indexes local repositories.
- **update**
  Updates the local index from a remote ZIP file.

## `layout-conf`
Tools to manipulate `metadata/layout.conf` values.

Usage: `g2 layout-conf list`
Usage: `g2 layout-conf <element> [get|set|unset]`

# EXAMPLES

Download an archive, calculate SHA256 in addition to the defaults, and update the Manifest in the current directory:
    $ g2 manifest -sha256=true upsert-from-url https://example.com/software-1.0.tar.gz software-1.0.tar.gz .

Add a maintainer to `metadata.xml`:
    $ g2 metadata -maintainer-add "dev@example.com:Developer Name:person" .

Install a package into a specific category, including extra files:
    $ g2 overlay ebuild install -category sys-apps my-app-1.0.ebuild . -- app.conf app.service

Generate a static HTML site for the overlay:
    $ g2 overlay site generate -out my_site_dir .

Serve the generated static site locally:
    $ g2 site serve -port 8080 .

Discover and populate USE flags automatically:
    $ g2 use discover .

Generate the ebuild cache for the current overlay:
    $ g2 cache generate .
